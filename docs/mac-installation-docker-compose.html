Überblick

NewsAtlas kann komplett auf dem eigenen Mac laufen — ohne Server, ohne Cloud-Abo. Dafür wird Docker Compose verwendet: Ein einziger Befehl startet vier Bausteine gleichzeitig in isolierten Containern:

BausteinAufgabe
dbPostgreSQL-Datenbank — speichert alle Daten
backendDie eigentliche NewsAtlas-Anwendung (API, News-Verarbeitung, KI-Zusammenfassung)
frontendDas Web-Dashboard, das im Browser angezeigt wird
caddyReverse-Proxy — stellt alles gebündelt unter einer einzigen Adresse bereit

Diese Anleitung führt jeden einzelnen Schritt im Terminal vor. Jeder Befehl kann einfach kopiert und eingefügt werden — es ist kein Programmierwissen nötig.

Schritt 1 — Docker Desktop installieren

1Docker Desktop herunterladen und installieren

Docker Desktop ist die Software, die die Container auf dem Mac ausführt. Sie ist kostenlos für den privaten Gebrauch bzw. für kleine Firmen.

  1. Offizielle Download-Seite öffnen: https://www.docker.com/products/docker-desktop/
  2. Die passende Version herunterladen: Mac mit Apple-Chip (M1/M2/M3/M4) oder Mac mit Intel-Chip — welcher Chip verbaut ist, steht unter Apple-Menü → Über diesen Mac.
  3. Die heruntergeladene Docker.dmg-Datei öffnen und Docker in den Programme-Ordner ziehen.
  4. Docker Desktop aus dem Programme-Ordner starten. Beim ersten Start werden Berechtigungen abgefragt (Systemeinstellungen) — diese bestätigen.
  5. Warten, bis das Wal-Symbol oben in der Menüleiste ruhig steht (kein Animations-Icon mehr) — das bedeutet, Docker läuft.
Prüfen, ob es funktioniert hat Terminal öffnen (siehe Schritt 2) und docker --version eingeben. Erscheint eine Versionsnummer statt einer Fehlermeldung, ist alles bereit.

Schritt 2 — Terminal öffnen

2Terminal-App öffnen

Alle folgenden Befehle werden in der Terminal-App eingegeben, die auf jedem Mac vorinstalliert ist.

  1. Cmd + Leertaste drücken (Spotlight-Suche öffnet sich)
  2. Terminal eintippen und Enter drücken

Ein Fenster mit einer Befehlszeile öffnet sich — hier werden die Befehle aus dieser Anleitung eingefügt.

Schritt 3 — NewsAtlas herunterladen

3Projektordner auf den Mac holen

Im Terminal an eine Stelle wechseln, an der der Ordner abgelegt werden soll (z. B. der eigene Benutzerordner), und das Projekt herunterladen:

# In den eigenen Benutzerordner wechseln
cd ~

# NewsAtlas herunterladen
git clone https://github.com/mimiron01/newsatlas.git

# In den neuen Ordner wechseln
cd newsatlas
Falls git noch fehlt macOS fragt beim ersten Ausführen von git automatisch, ob die "Command Line Developer Tools" installiert werden sollen. Einfach auf "Installieren" klicken und das Ende abwarten (wenige Minuten), danach den git clone-Befehl erneut ausführen.

Schritt 4 — Konfigurationsdatei anlegen

4.env-Datei aus der Vorlage erstellen

NewsAtlas wird über eine Datei namens .env konfiguriert (Zugangsdaten, Passwörter, API-Schlüssel). Sie wird aus einer mitgelieferten Vorlage erzeugt: 

cp .env.example .env
chmod 600 .env

Der zweite Befehl sorgt dafür, dass nur der eigene Benutzer die Datei lesen kann — sie enthält gleich echte Passwörter.

Schritt 5 — Geheime Schlüssel erzeugen

5Zwei zufällige Passwörter generieren

Zwei Werte in der .env-Datei müssen durch zufällige, sichere Zeichenketten ersetzt werden. Der Mac kann diese direkt erzeugen: 

# Datenbank-Passwort erzeugen
openssl rand -hex 24

# Anmelde-Schlüssel (JWT_SECRET) erzeugen
openssl rand -hex 32

Beide Befehle geben je eine lange Zeichenkette aus (z. B. 7f3a9c...). Diese einfach mit der Maus markieren und kopieren (Cmd + C) — sie werden im nächsten Schritt gebraucht.

Beide Befehle einzeln ausführen Für POSTGRES_PASSWORD und JWT_SECRET jeweils eine eigene, unterschiedliche Zeichenkette verwenden — nicht denselben Wert zweimal einsetzen.

Schritt 6 — Konfigurationsdatei ausfüllen

6.env im Texteditor öffnen und Werte eintragen

Die Datei lässt sich mit TextEdit direkt aus dem Terminal öffnen:

open -e .env

In der Datei folgende Zeilen suchen und die Werte anpassen (der Rest kann unverändert bleiben):

ZeileWert eintragen
DOMAIN=so lassen wie sie ist: localhost (nur für lokale Nutzung auf dem eigenen Mac)
POSTGRES_PASSWORD=die erste in Schritt 5 erzeugte Zeichenkette einfügen
JWT_SECRET=die zweite in Schritt 5 erzeugte Zeichenkette einfügen
SIGNUP_INVITE_CODE=ein selbst ausgedachtes Codewort, z. B. proair2026 — wird gleich zur Registrierung gebraucht
NEWSAPI_API_KEY=Schlüssel aus Schritt 7
MISTRAL_API_KEY=Schlüssel aus Schritt 7

Datei speichern (Cmd + S) und TextEdit schließen.

Optional, aber nicht zwingend für den Start NEWSDATA_API_KEY (zusätzliche Nachrichtenquelle) und die SMTP_*-Zeilen (für den täglichen E-Mail-Digest) können leer bleiben und später über die Einstellungen in der Web-Oberfläche nachgetragen werden.

Schritt 7 — API-Schlüssel besorgen

7Kostenlose/kostenpflichtige Zugänge bei den News- und KI-Anbietern anlegen

NewsAtlas benötigt zwei externe Dienste, um Nachrichten zu laden und mit KI zusammenzufassen:

  • NewsAPI.org (Nachrichtenquelle) — Konto anlegen auf newsapi.org/register, danach den angezeigten API-Key kopieren.
  • Mistral AI (KI-Zusammenfassung) — Konto anlegen auf console.mistral.ai, dort unter "API Keys" einen neuen Schlüssel erzeugen und kopieren.

Beide Schlüssel wie in Schritt 6 beschrieben in die .env-Datei eintragen.

Schritt 8 — NewsAtlas starten

8Alle Container bauen und starten

Zurück im Terminal (im newsatlas-Ordner) folgenden Befehl ausführen:

docker compose up --build -d

Beim ersten Mal dauert das einige Minuten, da die Software heruntergeladen und zusammengebaut wird. Der Fortschritt wird im Terminal angezeigt.

Wenn der Befehl fertig ist, prüfen, ob alles läuft:

docker compose ps

Alle vier Dienste (db, backend, frontend, caddy) sollten als running bzw. healthy angezeigt werden.

Schritt 9 — Erste Anmeldung im Browser

9NewsAtlas im Browser öffnen

  1. Einen Browser öffnen (z. B. Safari oder Chrome) und https://localhost aufrufen.
  2. Der Browser zeigt eine Warnung wie "Diese Verbindung ist nicht privat" oder "Zertifikat nicht vertrauenswürdig" — das ist bei lokaler Nutzung ohne echte Domain normal und unbedenklich. Auf "Erweitert" bzw. "Details einblenden" klicken und danach "Trotzdem fortfahren" wählen.
  3. Auf der Anmeldeseite "Registrieren" wählen.
  4. E-Mail-Adresse, ein selbstgewähltes Passwort sowie den in Schritt 6 vergebenen SIGNUP_INVITE_CODE eingeben.
  5. Nach der Registrierung ist man direkt im Dashboard und kann Zielunternehmen anlegen.

Im Alltag: stoppen, neu starten, aktualisieren

AktionBefehl (im newsatlas-Ordner ausführen)
NewsAtlas anhaltendocker compose down
NewsAtlas wieder startendocker compose up -d
Live-Protokoll ansehendocker compose logs -f
Nur das Backend-Protokoll ansehendocker compose logs -f backend
Auf eine neue Version aktualisierengit pull gefolgt von docker compose up --build -d
Achtung — Datenverlust docker compose down -v löscht zusätzlich alle gespeicherten Daten (Datenbank-Inhalt, Konten, Zielunternehmen, gesammelte Artikel) unwiderruflich. Für ein normales Stoppen immer docker compose down ohne -v verwenden.

Fehlerbehebung

"Cannot connect to the Docker daemon" Docker Desktop läuft nicht. Die App aus dem Programme-Ordner starten und warten, bis das Wal-Symbol in der Menüleiste ruhig steht, dann den Befehl erneut ausführen.
Fehlermeldung "POSTGRES_PASSWORD must be set" Die .env-Datei wurde nicht korrekt ausgefüllt oder es wurde vergessen, sie aus der Vorlage zu kopieren (Schritt 4). Schritt 4 und 6 erneut prüfen.
Backend startet nicht, Protokoll zeigt "Refusing to start ... insecure configuration" JWT_SECRET oder SIGNUP_INVITE_CODE in der .env-Datei sind noch leer oder zu kurz. Schritt 5 und 6 erneut durchgehen, Datei speichern und docker compose up --build -d erneut ausführen.
Port 80/443 bereits belegt Meldungen wie "port is already allocated" bedeuten, dass ein anderes Programm auf dem Mac bereits diese Ports verwendet (z. B. ein anderer lokaler Webserver). Das andere Programm beenden und docker compose up --build -d erneut ausführen.
"Ungültiger Einladungscode" bei der Registrierung Der eingegebene Code muss exakt (Groß-/Kleinschreibung beachten) mit dem Wert von SIGNUP_INVITE_CODE in der .env-Datei übereinstimmen.

Wichtiger Sicherheitshinweis

Nur für die lokale Nutzung auf dem eigenen Mac gedacht Diese Anleitung richtet DOMAIN=localhost ein und ist ausschließlich für den persönlichen Gebrauch auf dem eigenen Rechner bestimmt (z. B. für eine Präsentation oder zum Ausprobieren). Der Rechner sollte dabei nicht direkt aus dem Internet erreichbar gemacht werden (z. B. über Portweiterleitung im Router). Für einen öffentlich erreichbaren Betrieb mit echter Domain und automatischem HTTPS-Zertifikat siehe den Abschnitt "Deploying with Docker Compose" in der Haupt-README.md sowie docs/security-review.html.
